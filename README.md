## ⚙️ STEP 1 — Backend (FastAPI)

### Prerequisites
- Python 3.10+ installed
- Terminal / Command Prompt

### Setup & Run

```bash
# 1. Enter backend folder
cd backend

# 2. (Recommended) Create virtual environment
python -m venv venv

# On Windows:
venv\Scripts\activate

# On Mac/Linux:
source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Start the server
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### ✅ Verify it works
Open your browser: http://localhost:8000

You should see:
```json
{"status": "ok", "message": "LocalBiz API is running ✅"}
```

### API Docs (auto-generated by FastAPI)
- Swagger UI:  http://localhost:8000/docs
- ReDoc:       http://localhost:8000/redoc

### Available Endpoints

| Method | Endpoint     | Description              |
|--------|-------------|--------------------------|
| GET    | /           | Health check             |
| POST   | /chat       | Chatbot response         |
| POST   | /lead       | Save contact form lead   |
| GET    | /analytics  | View analytics data      |

### Test with curl

```bash
# Test chatbot
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "hello"}'

# Test lead capture
curl -X POST http://localhost:8000/lead \
  -H "Content-Type: application/json" \
  -d '{"name": "John", "email": "john@test.com", "message": "I am interested"}'

# View analytics
curl http://localhost:8000/analytics
```

### Data Storage
- `leads.csv` — saved in the backend folder automatically
- `chats.csv` — saved in the backend folder automatically

## 🌐 STEP 2 — Website

### Option A: Open locally (quickest)
1. Make sure the backend is running on port 8000
2. Double-click `website/index.html` in your file explorer
3. The chatbot and contact form will work immediately

### Option B: Serve via Python (recommended)
```bash
cd website
python -m http.server 3000
# Open: http://localhost:3000
```

### Changing the API URL
Open `index.html` and find this line near the bottom:
```javascript
const API_BASE = 'http://localhost:8000';
```
Change it to your server's IP or domain when deploying.

### WhatsApp Number
Search for `wa.me/1234567890` in `index.html` and replace with your number:
```
https://wa.me/YOUR_COUNTRY_CODE_AND_NUMBER
```
Example: `https://wa.me/447911123456` for UK number +44 7911 123456

## 📱 STEP 3 — Flutter App

### Prerequisites
- Flutter SDK 3.10+ installed → https://docs.flutter.dev/get-started/install
- Android Studio or VS Code with Flutter extension
- Android Emulator or physical device

### Setup & Run

```bash
# 1. Enter Flutter folder
cd flutter_app

# 2. Install dependencies
flutter pub get

# 3. Run on Android emulator (uses 10.0.2.2 → localhost)
flutter run

# Run on physical device (update IP first — see below)
flutter run
```

### ⚠️ IMPORTANT: Setting the correct API URL

Open `lib/controllers/chat_controller.dart` and update `_baseUrl`:

```dart
// Android Emulator → localhost
static const String _baseUrl = 'http://10.0.2.2:8000';

// iOS Simulator → localhost
static const String _baseUrl = 'http://localhost:8000';

// Physical Android/iOS device → your computer's local IP
static const String _baseUrl = 'http://192.168.1.XXX:8000';
```

**Finding your computer's local IP:**
- Windows: Run `ipconfig` → look for IPv4 Address
- Mac/Linux: Run `ifconfig` → look for inet address

**Make sure both your phone and computer are on the same WiFi network.**

### Android Network Permission
The file `android/app/src/main/AndroidManifest.xml` already includes the internet permission by default in Flutter. If you get network errors, add this inside `<manifest>`:
```xml
<uses-permission android:name="android.permission.INTERNET"/>
```

## 🔗 How Everything Connects

```
Website (index.html)
  ├── /chat  → FastAPI → returns bot reply → shown in chat UI
  └── /lead  → FastAPI → saved to leads.csv

Flutter App
  └── /chat  → FastAPI → returns bot reply → shown in chat UI

Analytics
  └── GET /analytics → FastAPI → reads leads.csv + chats.csv → returns JSON
```

## 🚀 Deploying to Production

### Backend (FastAPI)
Deploy to any of these free/cheap options:
- **Railway.app** — push to GitHub, auto-deploys
- **Render.com** — free tier available
- **VPS (DigitalOcean/Hetzner)** — full control

After deploying, update `API_BASE` in `index.html` and `_baseUrl` in Flutter to your live URL.

### Website
- Upload `index.html` to **Netlify**, **Vercel**, or any web host
- Or use GitHub Pages (free)

## ❓ Troubleshooting

| Problem | Fix |
|---------|-----|
| CORS error in browser | Make sure FastAPI is running; CORS is already set to `allow_origins=["*"]` |
| Flutter can't connect | Check `_baseUrl` in `chat_controller.dart` |
| CSV not created | Backend creates files automatically on first run |
| Port 8000 in use | Run with `--port 8001` and update all API URLs |
| `uvicorn` not found | Activate virtual environment first |
